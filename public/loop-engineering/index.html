Repository Harness5-minute pitch
Open by naming the category: not another coding-agent demo, but a way to make repeated agent work safe enough to operate inside real repositories.

problem.md

Most repositories are not ready for autonomous agents.

Agents can write code

But the repo rarely tells them what matters, what is risky, or what proof will count.

Intent is scattered

Scope, constraints, validation, and decisions live across chat, memory, files, and people.

Autonomy becomes risky

Unclear boundaries create weak proof, hard review, and fragile follow-up loops.

01 / Problemscope · proof · review
The pain is not that agents are weak. The pain is that the repository does not yet behave like a control system for those agents.

demo.script

Demo goal: show one complete loop.

1. Open the controllerharness-symphony web, then scan hierarchy and dependency graph.
2. Start a Ready taskOnly runnable work can enter the active loop.
3. Watch Codex eventsLive App Server events show execution, not a hidden black box.
4. Review evidenceSummary, result, validation, changed files, changeset, PR state, and logs.
5. Mark PR mergedThe human confirms the external review outcome.
6. Approve syncThe task moves to Done only after proof and sync close the loop.
